er_yaml = """
entities:

  key: LPID
  attributes:

dbman = DuckDbMan.fromMem()

sql = """
CREATE TABLE Gebaeude (
  GID INTEGER NOT NULL,
  Bez VARCHAR(50) NOT NULL,
  PRIMARY KEY (GID)
);
CREATE TABLE Lehrperson (
  LPID INTEGER NOT NULL,
  Name VARCHAR(100) NOT NULL,
  EMail VARCHAR(100),
  GID INTEGER,
  PRIMARY KEY (LPID),
  CONSTRAINT fk_hat_buero_in FOREIGN KEY (GID) REFERENCES Gebaeude (GID)
);
"""
dbman.execute(sql)

Ausführung des SQL-Codes zum Einfügen von Daten.

sql = """
insert into Gebaeude values (1, 'C');
insert into Lehrperson values (1, 'Claßen', 'ingo.classen@htw-berlin.de', 1);
insert into Lehrperson values (2, 'Kempa', 'martin.kempa@htw-berlin.de', 1);
"""
dbman.execute(sql)

Ausführen und Anzeigen der Ergebnisse einer Abfrage.

sql = """
select g.Bez as Gebaeude, lp.Name as Lehrperson, lp.EMail
from Gebaeude g
     join Lehrperson lp on g.GID = lp.GID

Datengenerierung mit der KI

Erstellung eines Relationship-Managers (relman). Der wir benötigt, um das Datenbankschema im YAML-Format zu definieren. 

er_yaml = """
entities:
- entityname: Modul
  key: MID
  attributes:
  - attributename: Bez
    type: VARCHAR(50)
valuelists:
- valuelistname: Modulart
  values:
  - Pflicht
  - Wahlpflicht
  - Wahl
  many_to_one_from_entities:
  - sourceentity: Modul
"""
erman = ERSchemaMan.fromStr(er_yaml)
relman = RelSchemaMan.fromAST(erman.get_rel())

Anlegen einer DataGenConversation. Das ist die Grundlage für die Prompt-Generierung.

datagen_conv = DataGenConversation()

Setzen des Datenbankschemas. Hier kommt die YAML-Generierung von relman ins Spile. 

datagen_conv.set_database_schema(relman.get_yaml())

Einfügen einer Datengenerierungs-Nachricht in die Konversation. Hier werden die Anzahl der zu generierenden Datensätze pro Tabelle und ggfs. auszuschließende Tabellen angegeben. 

datagen_conv.insert_datagen_message(
    no_of_records_per_table=5, 
    exclude_tables=["Modulart"]
    ))

Erzeugen des Prompts für die KI der Wahl (ChatGPT, Claude, usw.)
